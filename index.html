MODULE 02

BIẾN & KIỂU DỮ LIỆU

Stack Memory, Advanced Types & Casting

BỘ NHỚ STACK & BIẾN

Các biến cục bộ (Local variables) được lưu trữ trên Stack.

  • Tốc độ: Rất nhanh.
  • Cơ chế: LIFO (Last In, First Out).
  • Tự động: Tự giải phóng khi ra khỏi phạm vi hàm `{ }`.
{
    int a = 10; // Push vào Stack
    int b = 20; // Push đè lên a
} 
// b bị hủy trước, rồi đến a (Pop)

Stack Visualization

int b Value: 20
int a Value: 10
... ...

ÉP KIỂU (TYPE CASTING)

Chuyển đổi giá trị từ kiểu dữ liệu này sang kiểu dữ liệu khác.

Implicit (Ngầm định)

Compiler tự động thực hiện, thường là từ kiểu nhỏ sang kiểu lớn.

int x = 10;
double y = x; 
// y = 10.0

Explicit (C-Style)

Ép kiểu tường minh theo phong cách C cũ.

double pi = 3.14;
int z = (int)pi;
// z = 3 (mất phần thập phân)

Static Cast (C++)

An toàn hơn, kiểm tra tại thời điểm biên dịch.

double pi = 3.14;
int z = static_cast<int>(pi);
MODULE 03

LUỒNG ĐIỀU KHIỂN

Control Flow, Switch-Case & Jumps

SWITCH & FALL-THROUGH

Dùng switch khi so sánh 1 biến với nhiều giá trị cụ thể.

Cảnh báo Fall-through

Nếu QUÊN break, chương trình sẽ tiếp tục chạy xuống các case bên dưới!

switch (option) {
    case 1:
        runGame();
        break; // Thoát switch
    case 2:
        showInfo();
        // QUÊN BREAK Ở ĐÂY!
    default:
        exit(); // Case 2 sẽ chạy luôn dòng này
}

LOOPS & JUMP STATEMENTS

For / While

Dùng For khi biết số lần lặp. Dùng While khi lặp theo điều kiện.

Break

Thoát khỏi vòng lặp NGAY LẬP TỨC.

if (found) break;

Continue

Bỏ qua phần còn lại của lần lặp hiện tại, nhảy sang lần kế tiếp.

if (i % 2 == 0) continue;
MODULE 04

HÀM (FUNCTIONS)

References, Overloading & Recursion

THAM TRỊ vs THAM CHIẾU

Pass by Value (Tham trị)

Copy giá trị. Biến gốc KHÔNG đổi.

void inc(int n) {
    n++; 
}
// Gọi inc(x), x vẫn như cũ

Pass by Reference (Tham chiếu)

Dùng địa chỉ. Biến gốc THAY ĐỔI.

void inc(int &n) {
    n++;
}
// Gọi inc(x), x tăng lên 1

OVERLOADING & ĐỆ QUY

Function Overloading

Nhiều hàm CÙNG TÊN nhưng KHÁC tham số.

int add(int a, int b) { ... }
double add(double a, double b) { ... }

Recursion (Đệ quy)

Hàm gọi lại chính nó. Cần điều kiện dừng (Base case).

int fact(int n) {
    if (n <= 1) return 1;
    return n * fact(n - 1);
}
MODULE 05

MẢNG & CHUỖI

Contiguous Memory, C-String vs std::string

MẢNG: BỘ NHỚ LIÊN TIẾP

Các phần tử mảng nằm sát nhau trong RAM. Truy cập cực nhanh qua Index.

addr: 100
A[0]
addr: 104
A[1]
addr: 108
A[2]
addr: 112
A[3]

Lưu ý: Mảng C++ không kiểm tra giới hạn (No Bounds Checking). Truy cập A[100] khi size=4 sẽ gây lỗi hoặc hành vi không xác định.

C-STRING vs STD::STRING

C-Style String

Mảng ký tự kết thúc bằng \0. Khó dùng, dễ lỗi.

char s[] = "Hello";
// Kích thước cố định
// Phải dùng strcpy, strcat...

std::string (Nên dùng)

Lớp đối tượng mạnh mẽ, tự động quản lý bộ nhớ.

#include <string>
string s = "Hello";
s += " World"; // Nối chuỗi dễ dàng
cout << s.length();
MODULE 06

CON TRỎ (POINTERS)

Memory Addresses, Dynamic Allocation & Leaks

BẢN CHẤT CỦA CON TRỎ

Con trỏ là một biến đặc biệt, giá trị của nó là ĐỊA CHỈ của biến khác.

  • & (Address-of): Lấy địa chỉ của biến.
  • * (Dereference): Truy cập giá trị tại địa chỉ đó.
int a = 10;
int* p = &a; // p trỏ tới a

cout << p;  // In ra 0x7ffd...
cout << *p; // In ra 10

CẤP PHÁT ĐỘNG & MEMORY LEAK

Heap Memory

Bộ nhớ Heap lớn hơn Stack, sống cho đến khi bị xóa.

// Cấp phát 1 số nguyên trên Heap
int* p = new int; 
*p = 100;

// Cấp phát mảng động
int* arr = new int[500];

Memory Leak (Rò rỉ)

Quên giải phóng bộ nhớ Heap sẽ gây tràn RAM.

delete p;      // Xóa biến đơn
delete[] arr;  // Xóa mảng

// Quan trọng: p = nullptr sau khi xóa
p = nullptr;
